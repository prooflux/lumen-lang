# Lumen Integrated Design (Synthesis v1)

Status: synthesized from the `lumen-design-deepening` multi-agent workflow (run wf_84ab3375-9b8): 17 design dimensions, each adversarially critiqued, then integrated and gap-checked. This document is the integrated design of record. Where it conflicts with an earlier doc, this wins, and the affected doc is on the update list in `RISKS_AND_OPEN_PROBLEMS.md`.

## The organizing principle

Lumen is not 17 subsystems. It radiates from **one mechanism and one artifact**, and everything else is a consequence:

- **The unforgeable capability value** is the single mechanism for effects, authority, purity, and determinism classification. There is no separate effect system, no `async`/`unsafe`/`throws` keyword family, no ambient authority. A function's effect row is *derived* from the capability types it transitively uses, never hand-written. Zero capability parameters means the compiler proves the function pure.
- **The canonical structured Diagnostic** is the single artifact every tool, pass, and runtime emits. One schema, schema-versioned, typed arguments, the human message rendered downstream from a registry template (not carried in the machine stream). No dimension forks it.

Every cross-dimension conflict the workflow surfaced reduced to one of **four root resolutions**:

1. **Determinism is one compiler-enforced contract**, referenced (not re-asserted) by memory, concurrency, debug, IR, security, verification, const-eval, FFI, and bootstrap. See `DETERMINISM_CONTRACT.md`.
2. **The Diagnostic is one generated schema-versioned type**, shared by errors, debug, IR, tooling, stdlib, security, verification, and governance, with no per-dimension fork. See `DIAGNOSTIC_SCHEMA.md` (to be authored as an executable schema).
3. **The capability is the one effect/authority/purity/determinism mechanism**: author-written effect and determinism keywords are abolished in favor of derived rows; capability-kindedness is transitive through aggregates and closures; sub-capabilities are nominal (not structural); attenuation is branded into a distinct static type; and an affine, scope-bounded non-escape invariant ties memory, concurrency, security, verification, and interop together.
4. **Legacy smells are uniformly purged or re-earned.** The authorship benchmark is the empirical court that re-earns any retained borrowing rather than asserting it. Removed or replaced: errno-as-flag, semver-as-identity, UTF-16 spans, implicit `From` at error propagation, casing-as-namespace, the abbreviation allowlist, Eiffel `requires`/`ensures`/`old` vocabulary, insertion-order maps, and the Rust differential oracle in the bootstrap trust path.

## Per-dimension decisions (post-critique)

Each line is the final decision after adversarial review. "Was weak" marks dimensions the critique materially repaired.

- **Syntax.** Fully-delimited, statement-shell over expression-core grammar, normalized at a CST layer to exactly one canonical form, enforced by a zero-option total formatter with the proven invariant `format(parse(format(x))) == format(x)`. Field access and method application are first-class productions in a left-factored, non-left-recursive grammar (the LL(1) claim is restated honestly as left-factored LL(1) plus a documented Pratt-style postfix layer). The dual block-value spelling is collapsed: the trailing expression is the block value, `give` is removed, and early exit unifies into one labeled block-escape. Operator precedence across classes is replaced by a total, proven `requiresParens` matrix (the formatter inserts parens); same-class chains are bare; `a < b < c` is defined as a conjunction, not an error. Value vs type is by declaration keyword, never by casing. Call style is uniform; the formatter canonicalizes argument order to declaration order. Syntax diagnostics carry typed expected/actual; recovery is delimiter-stack panic-mode with one diagnostic per fault.
- **Types.** Bidirectional, local-only checking (no global Hindley-Milner, no cross-statement generalization), so every error is co-located with its annotation. Mandatory annotations on every top-level binding, parameter, and public field. One sum (nominal ADT), one product (record), one abstraction (trait). Rank-1 polymorphism plus the minimal higher-kinded extension that trait associated types and effect rows actually require (F-omega dropped as identity). Row polymorphism retained because records and effect rows share one row-unification engine. No subtyping, variance, coercion, or top type. Refinement types are first-class and erased, discharged by a deterministic SMT backend. Types are a queryable content-addressed fact base.
- **Memory.** Inferred-ownership Perceus-style compiler-inserted reference counting; no source lifetimes, no `&`/`&mut`, no smart-pointer zoo, no `clone()` in source. Default value world is structurally acyclic; cycles live only in an explicit capability-gated `Graph` arena, and mutating that arena is explicitly impure. Drop placement is specified over real control flow (branches, loops, early return, match, partial moves). Region escape is reject-with-fix, never a silent deep copy. A `Ref` into an arena carries a static phantom-region brand tying it to its arena. A unified affine (linear-on-request) `Resource` layer makes "forgot to close" a compile error, with the consume obligation enforced on every exit path including error propagation.
- **Concurrency** (surface deferred, semantics specified). Two layers over capabilities: deterministic data-parallelism over owned or frozen data (bit-identical across thread counts, explicitly not equal to a sequential fold for non-associative floating point), and structured concurrency with a nursery capability and move-only typed channels (data-race-freedom is a type property), plus a frozen shared-immutable capture for read-only fan-out. The scheduler is a pure function of the program plus a recorded capability/select log. Deadlock is a runtime diagnostic with a replay transcript (not an unbacked compile-time guarantee) until session types are added. A typed, replay-preserving `relaxed` mode exposes the determinism-versus-throughput tradeoff honestly.
- **Effects and capabilities** (the spine, strongest dimension). As in the organizing principle. Handlers (algebraic-effect interception) provide capabilities at a delimited scope for deterministic test mocking; resumption is one-shot by default, and multi-shot taints the row with a marker. Soundness rests on the affine, scope-bounded capability-non-escape invariant, specified here, not deferred.
- **Errors.** One recoverable mechanism: `Result[T, E]` propagated by a single explicit, conversion-visible operator (no implicit `From`). Panic-ability is visible in the effect summary (`panics: none | set`); the supervision boundary is gated by an `Isolate` capability, so it appears in the signature and is not an exception in disguise. Overflow and divide-by-zero share the `Result` plane for faults reachable from untrusted input, so a single adversarial integer cannot abort a process; named wrapping/saturating/checked variants exist for hot paths. The auto-apply fix tier excludes signature- or type-changing edits.
- **Debuggability.** Built on the structural fact that nondeterminism is a closed, enumerable set (every effect is a capability seam). The tape is a seam log (size proportional to bytes crossing capability boundaries); pure code is recomputed, not stored; reverse execution uses snapshots plus a Perceus-reuse undo-log; `why`-provenance runs over a provenance-preserving MIR (every pass must emit provenance-rewrite edges). "Free" is retracted: the Determinism Contract is the enforced precondition. The two-backend bit-exact problem is resolved by a canonical replay-floating-point lowering both backends must hit, with a `TAPE_BACKEND_MISMATCH` diagnostic. Eliminated values get a typed `Eliminated(pass, span)` terminal cause. The agent-facing interface is a content-addressed composable query algebra, not human step-debugging.
- **Compiler and IR.** One demand-driven, content-addressed query engine over CST/AST/HIR/MIR/LIR. The stable external interface (diagnostics, provenance, repro manifest) is split from the evolvable internal MIR. Backend determinism is backed (pinned floating point and const-eval, deterministic codegen ordering, vendored libm, cross-machine object-byte conformance). Parallel-agent safety uses behavioral golden corpora and a composition gate, not schema hashes alone. Every MIR optimization pass emits provenance-rewrite edges. Binary IR is canonical; text is a derived view.
- **Tooling.** One binary `lumen`, one Tool Protocol, one Universal Diagnostic Object that is the canonical Diagnostic (no fork). UTF-16 spans removed; canonical verbs restored (`run.tape`, `debug/why`, `type`/`effects`/`callers`/`ast`); Tool Protocol responses are deterministic. Fixes split into a proven-safe `auto` tier and a separate `refactor` channel, with the shipping property that any auto-fix still type-checks. Tests are pure functions taking explicit capabilities with deterministic seeding and replay. Packages are content-addressed with reproducible builds; semver is registry-only metadata, content hash is identity.
- **Standard library.** `core` (capability-free, pure, total) versus `std` (capability-gated). A naming law with a closed verb vocabulary, validated by the authorship benchmark before freezing (the `reduce`/`fold` synonym removed, the abbreviation allowlist dropped unless a full word is not a stronger model prior). Floating point pinned across backends; NaN/Inf non-propagating by construction. Map/set iteration order is canonical (sorted) by default, with insertion order as an explicit type. Capability granularity and closure-capture purity precisely defined: a function is impure iff a capability is reachable via a parameter or a closure capture. UFCS removed; module-qualified is the only canonical form.
- **AI-authorship feedback loop.** The compiler is bidirectional, recording how the model reacts to diagnostics and gating evolution on measured authorability. Decisive critique fixes: the merge-blocking regression gate is deterministic (only open-weight, locally-pinned, content-addressed model snapshots may block; closed frontier models are advisory only); surprisal is demoted to a localization signal with proper-scoring calibration on a model-uninfluenceable held-out set and rotating-secret metrics; grammar changes are gated behind cross-model-correlation tests so the current cohort's shared blind spots cannot be encoded into the grammar (model-specific friction routes to skill guidance, not grammar); authorability can never override a correctness fix.
- **Security and supply chain.** Authority is a first-class unforgeable type-level value; the same object-capability model is the type system, the registry, and the runtime sandbox. Transitive capability-kindedness through aggregates and closures (without it, no-ambient-authority was false). Revocation via a membrane-identity mechanism reconciled with Perceus value semantics. Deterministic fuel and allocation metering (no wall-clock traps). Resource confinement (hosts, paths) is in the registry manifest and lockfile. A dedicated, signature-visible `Declassify` capability makes prompt-injection fail-closed. Content-addressed packages, zero install-time execution, transparency-log publishing.
- **Metaprogramming.** No syntactic macros, no custom syntax: the grammar is frozen per version, so the model's surface distribution is stationary. Four typed mechanisms: const-evaluation, `derive` (the only codegen-from-types), read-only compile-time reflection returning data, and typed staged templates (`Code[T]`) that must re-typecheck as ordinary code with a structured expansion trace. Totality is enforced by a discharged termination measure (fuel is only an error budget). Const-eval determinism is pinned. "No operator overloading" is restated as "no user-defined operators" (overloading a fixed closed set via trait instances is allowed).
- **Interop and FFI.** No inline FFI and no C in the language: a declarative `extern bridge` manifest plus a compiler-generated audited thunk is the only place unsafe exists, gated by a capability. Bridge results carry a propagating nondeterminism taint; deterministic-replay is separated from deterministic-execution. Callbacks into Lumen are specified (capability-carrying trampoline, lifetime, reentrancy). ABI lowering is derived from the codegen backend as the authoritative oracle plus a debug ABI probe (no hand-maintained tables). Owned returns carry an explicit exclusive-versus-borrowed annotation.
- **Verification** (was weak, most changed). SMT-discharged refinement types and effect-typed contracts, with one predicate AST shared by refinements, runtime checks, property tests, and provenance queries. The always-green ladder is removed: default verification fails below `proven`, and downgrades are explicit, ledgered, reason-bearing, and penalized by the authorship benchmark, so the AI is not incentivized to defeat verification. The wall-clock SMT timeout is replaced by a deterministic instantiation budget, and in-fragment versus out-of-fragment is a static syntactic decision. The Eiffel `requires`/`ensures`/`old` vocabulary is replaced by a single relational refinement over pre/return state. Totality on recursive pure functions requires a discharged termination measure. Property-based testing is one evidence mechanism alongside bounded-exhaustive.
- **Governance and community.** Apache-2.0 for the toolchain, CC0 for spec, corpus, and schemas. The conformance corpus is no longer the sole normative artifact: a formal mechanized operational semantics for the canonical IR is the normative ground, with the corpus as the conformance witness. The authorship-benchmark non-regression predicate is removed from the hard merge gate (advisory only). Anti-overfitting via a held-out private shard and metamorphic generators. The gate is structurally above the council via change-controlled, content-addressed, council-non-tunable baselines.
- **Bootstrap** (was weak). A two-seed, three-stage bootstrap to a self-hosting fixpoint with no legacy high-level language on the critical path: a tiny separately-verified WAT seed kernel interpreting Lumen-mu, then a Lumen-mu compiler, then self-compile to a fixpoint where the seed is discarded. The Rust differential oracle is replaced by an independently-authored non-legacy second seed for diverse double-compilation. `lc1 == lc2` is a stability check, not a correctness proof; correctness is conformance plus double-compilation convergence plus an optional mechanized mu-evaluator proof. The byte-identity gate runs only over a provably deterministic artifact (WASM or Cranelift-debug, never LLVM release), with the pinned WASM engine named in the trusted computing base.

## Cross-dimension conflict resolutions

1. **Memory vs concurrency vs determinism.** One Determinism Contract owns drop placement, fixed reduction trees, deterministic ordering, and fuel-based metering. Concurrency reuses Perceus uniqueness to prove chunk disjointness and moves values over channels (ownership transfer, not shared refcount); contended atomic refcount cost is disclosed and confined to the `Graph` arena.
2. **Effects vs types vs verification.** Capabilities are the single source. Determinism classification lives in the trusted runtime root (no author keyword). Effect rows and record rows share one unification engine. Purity makes refinements total specs; verification uses a deterministic instantiation budget and a static fragment decision, so the type-level verdict is machine-independent.
3. **Errors vs effects vs debug (the panic channel).** Panic-ability is a visible effect-summary component; the catch boundary is gated by an `Isolate` capability, so the abort channel is signature-visible, recordable as a seam, and not an invisible exception.
4. **Errors vs stdlib vs security (the arithmetic-fault plane).** Untrusted-reachable integer and division faults unify onto the `Result` plane; float NaN/Inf are non-propagating; range-typed integers turn overflow into a construction-time type error where applicable, removing the covert-timing channel.
5. **Metaprogramming vs clarity vs determinism.** The grammar is frozen per version (no library adds syntax); generative power is confined to `derive` and typed `Code[T]` that re-typechecks, recoverable via a structured expansion trace; const-eval determinism is pinned and totality enforced.
6. **Bootstrap vs zero-legacy vs security/determinism.** The Rust oracle is replaced by an independent non-legacy second seed; the byte-identity gate runs over a provably deterministic artifact; the pinned WASM engine is named in the trusted computing base.
7. **The Diagnostic across eight dimensions.** One generated, schema-versioned Diagnostic with typed args, byte spans, a fix array with separated auto/refactor tiers, and structural family+discriminant codes. Governance change-controls the schema; conformance verifies no fork.
8. **Feedback/governance vs determinism (the meta-conflict).** Stochastic LLM measurements are advisory, not deterministic merge gates; only locally-pinned open-weight snapshots may block; grammar changes are gated behind cross-model-correlation tests; authorability never overrides correctness.
